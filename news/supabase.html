Supabase 免费计划详细整理

一、免费计划特点

  • 数据库存储：500 MB
  • 文件存储：1 GB
  • 带宽：每天 5 GB 出站流量（不含缓存）
  • 每月活跃用户数（MAU）：最多 50,000
  • 边缘函数：每天最多 1,000 次调用，每秒最多 2 次调用
  • 项目数量：每个组织最多 2 个免费项目
  • 支持类型：社区支持
  • 自动暂停：项目 7 天未使用，将自动暂停
  • 适用场景：个人项目、MVP、学习和实验

注意：免费计划适用于非商业用途，如用于商业项目请遵守 Supabase 使用条款。

二、关键限制点

  • 项目数量限制：每个组织最多 2 个免费项目
  • 项目暂停：7 天未使用自动暂停
  • 功能限制：不支持自定义域名、备份、日志保留超过 1 天等高级功能
  • 边缘函数限制：每个项目最多 100 个边缘函数，最大内存 256MB，最大执行时间 150 秒
  • 邮件发送限制：每月最多发送 1,000 封邮件，超出需使用自定义 SMTP 服务

三、申请与激活步骤

  1. 访问官网：Supabase 官网
  2. 创建账户：使用 GitHub、Google 或邮箱注册
  3. 创建项目：
    • 登录后点击“New Project”
    • 选择“Free Plan”
    • 填写项目名称、数据库密码等信息
  4. 项目配置：配置数据库、认证、存储等服务
  5. 开始使用：通过 Supabase Dashboard 管理和开发项目

四、激活技巧与优化建议

  • 保持活跃：定期访问项目，避免 7 天未使用被暂停
  • 合理规划资源：在免费计划限制内进行开发，避免超出配额
  • 使用自定义 SMTP 服务：发送大量邮件时可绕过免费限制
  • 监控使用情况：通过 Dashboard 查看使用情况，及时调整
  • 利用社区支持：在 GitHub Discussions 或 Discord 寻求帮助

五、使用场景建议

场景类型 推荐理由
个人项目 免费计划提供足够资源，适合个人开发和学习
MVP 开发 快速搭建原型，验证产品概念，节省成本
教学与实验 用于教学演示和实验，便于快速部署和迭代
小型应用 适用于用户量较少的应用，如内部工具、个人博客等
